Prosesor (CPU)
clock_speed · core_count · nm_process
3.2
GHz · 8 core
Apa itu? Prosesor adalah "otak" HP kamu — semakin cepat dan banyak inti (core)-nya, semakin banyak pekerjaan yang bisa dilakukan sekaligus.

Hubungannya dengan app ini: Enkripsi AES-256 butuh kalkulasi berat. Prosesor dengan 8 inti @ 3.2 GHz bisa mengenkripsi file besar dalam hitungan milidetik — bukan menit.
Bayangkan dapur restoran: makin banyak koki (core) dan makin cekatan (GHz), makin cepat semua pesanan selesai.
RAM (Memori Kerja)
ram_gb · lpddr_version · bandwidth_gbps
8
GB LPDDR5
Apa itu? RAM adalah "meja kerja" HP — tempat data sementara diletakkan saat sedang diproses. Semakin besar, semakin banyak yang bisa dikerjakan tanpa lemot.

Hubungannya dengan app ini: Saat enkripsi file besar (foto, video, PDF), data harus masuk ke RAM dulu. Dengan 8 GB LPDDR5, file hingga 2 GB bisa diproses tanpa app crash atau freeze.
RAM itu seperti meja kerja: meja lebar = bisa buka banyak berkas sekaligus. Meja sempit = sering bolak-balik lemari arsip.
Secure Enclave / TEE
tee_enabled · keystore_hw · strongbox
✓
Hardware
Apa itu? Ini adalah "brankas dalam brankas" — chip keamanan terpisah di dalam HP yang menyimpan kunci enkripsi. Bahkan kalau HP-mu diretas, chip ini tetap aman.

Hubungannya dengan app ini: Kunci AES-256 kamu disimpan di Secure Enclave, bukan di memori biasa. Hacker tidak bisa mencurinya meski mereka punya akses root ke HP-mu.
Seperti brankas baja di dalam rumah yang sudah dikunci. Maling bisa masuk rumah, tapi brankas tetap tak tersentuh.
Kecepatan Storage
read_mbps · write_mbps · nvme_ufs
2.1
GB/s Read
Apa itu? Kecepatan baca-tulis storage (UFS 3.1 / NVMe) — seberapa cepat HP bisa mengambil atau menyimpan data dari memori internal.

Hubungannya dengan app ini: Enkripsi file 100 MB butuh baca + tulis sekaligus. Dengan 2.1 GB/s read speed, file selesai dienkripsi dalam <1 detik — bukan 10 detik seperti HP lama.
Bayangkan kecepatan mengangkut barang dari gudang ke meja kerja. Truk besar (UFS 3.1) vs gerobak (eMMC).
Akselerasi Kripto (HW)
aes_hw_accel · sha_engine · rng_hw
✓
Aktif
Apa itu? Chip modern punya "mesin khusus" untuk kalkulasi enkripsi — terpisah dari prosesor utama. Ini namanya AES Hardware Accelerator.

Hubungannya dengan app ini: Tanpa akselerasi HW, enkripsi memakan 80% CPU. Dengan akselerasi HW aktif, CPU hanya dipakai 8% — HP tetap mulus dipakai, baterai lebih hemat.
Seperti kalkulator khusus untuk matematika berat. CPU tidak perlu bekerja keras — ada "asisten" yang bantu ngitung.
Baterai & Termal
battery_mah · throttle_temp · tdp_mw
5000
mAh · 45°C
Apa itu? Kapasitas baterai (mAh) + batas suhu sebelum HP "ngerem" sendiri (thermal throttling). HP panas = performa turun otomatis.

Hubungannya dengan app ini: Enkripsi massal (100+ file sekaligus) butuh sesi panjang. Baterai 5000 mAh + manajemen termal bagus = proses selesai tanpa HP panas dan melambat di tengah jalan.
Seperti pelari maraton — bukan soal kecepatan awal, tapi stamina untuk selesaikan lomba tanpa kelelahan di km terakhir.
